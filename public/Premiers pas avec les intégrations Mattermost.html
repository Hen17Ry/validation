Premiers pas avec les intégrations Mattermost

Ben Hall

6 janvier 2020

Plate-forme

Mattermost est bien connu comme une plate-forme de messagerie flexible et open source. Mais ce qui le rend encore plus utile, c'est sa capacité à automatiser les connexions avec des robots et des webhooks et à se connecter à des applications externes. Ces automatisations et connexions sont appelées intégrations. 

De nombreuses intégrations sont disponibles dans le commerce via le répertoire d'intégrations . Mais il existe de nombreuses façons de créer la vôtre.

Par exemple, les webhooks assurent une communication légère entre les applications en temps réel et sont généralement utilisés pour envoyer des notifications d'événements. 

Les webhooks entrants permettent à Mattermost de recevoir des données d'applications externes et de les partager dans des canaux. Pour illustrer, disons que vous recevez des demandes d'assistance provenant de diverses sources. Ceux-ci pourraient tous être acheminés vers un canal où votre équipe peut en discuter et prendre les mesures nécessaires.

D'un autre côté, les webhooks sortants sont utilisés pour envoyer des données à d'autres applications et pour publier des réponses dans un canal. Vous les utilisez généralement conjointement avec des commandes slash. 

Les commandes Slash commencent par une barre oblique (c'est-à-dire /) et peuvent être utilisées à la fois pour des réponses automatisées internes et pour envoyer des requêtes HTTP à un service Web. Par exemple, vous souhaiterez peut-être améliorer le système d'assistance afin que votre équipe puisse saisir des commandes dans le canal pour enregistrer des actions ou attribuer ou clôturer des problèmes et que ces commandes soient envoyées vers une application externe.

De plus, le serveur Mattermost expose une API REST que vous pouvez appeler à partir d'applications plus avancées. Pour plus d’informations, consultez notre guide de démarrage sur les intégrations.

Maintenant, regardons quelques exemples. 

Le premier montre comment envoyer des messages de votre serveur Jenkins à votre serveur Mattermost. La seconde illustre comment Mattermost peut envoyer des messages à Jenkins.

Jenkins envoie des messages à Mattermost

La mise en place d'une intégration entre un serveur Jenkins CI et Mattermost implique simplement d'installer un plugin sur votre serveur Jenkins et d'ouvrir un point de terminaison webhook sur votre serveur Mattermost.

Pour ajouter un webhook entrant sur votre serveur Mattermost, accédez à Menu principal > Intégrations > Webhook entrant > Ajouter un webhook entrant .

Si l'option de menu pour les intégrations est manquante, dirigez votre administrateur vers ce guide pour l'activer. 

Vous vous retrouverez avec une URL générée pour vous, prête à envoyer des données vers et depuis Jenkins :

Ensuite, installez le plugin Mattermost Notification sur votre serveur Jenkins via l' interface utilisateur de Plugin Manager :

Configurez maintenant le point de terminaison du webhook Mattermost sur votre serveur Jenkins.

Depuis le tableau de bord Jenkins, accédez à Gérer Jenkins > Configurer le système > Paramètres globaux de notification Mattermost et collez l'URL du point de terminaison générée par le serveur Mattermost pour le webhook entrant que vous avez ajouté.

Vous devrez inclure l'adresse du serveur de build Jenkins pour tout lien de retour.

Configurez Jenkins pour envoyer des données à Mattermost. L'exemple le plus simple consiste à envoyer un message de test dans une couleur donnée sous forme de valeur hexadécimale :

mattermostSend(color: "#2A42EE", message: "Simple test message")

La capture d'écran ci-dessous montre un exemple de pipeline Jenkins pour créer une application Java Maven simple . L'exemple utilise l'approche de pipeline déclaratif, mais un script est intégré pour fournir plus de contrôle sur le moment où déclencher les événements liés à l'échec de la construction. 

Voici ce qui se passe :

  1. Nous envoyons un message, en bleu, à Mattermost, informant le canal que la build a commencé et fournissant le nom de la build, son numéro et un lien vers celle-ci.
  2. Nous commençons la construction de maven. C'est dans le bloc try afin que nous puissions détecter un échec.
  3. En cas d'échec, nous captons l'exception et définissons le résultat de la construction sur FAILURE.
  4. Dans le bloc final, nous utilisons le danger (rouge) et le bien (vert) pour envoyer le message d'échec ou de réussite approprié à Mattermost.
pipeline {
    agent {
        docker {
            image 'maven:3-alpine'
            args '-v /root/.m2:/root/.m2'
        }
    }
    stages {
        stage('Build') {
           steps {  
                script {
                    try {
                        mattermostSend (
                            color: "#2A42EE", 
                            message: "Build STARTED: ${env.JOB_NAME} #${env.BUILD_NUMBER} (<${env.BUILD_URL}|Link to build>)"
                        )                   
                        sh 'mvn -B -DskipTests clean package'
                    } catch(e) {
                        currentBuild.result = "FAILURE"
                    } finally {
                        if(currentBuild.result != "FAILURE") {
                            mattermostSend (
                                color: "danger", 
                                message: "Build FAILED: ${env.JOB_NAME} #${env.BUILD_NUMBER} (<${env.BUILD_URL}|Link to build>)"
                            )
                        } else {
                            mattermostSend (
                                color: "good", 
                                message: "Build SUCCESS: ${env.JOB_NAME} #${env.BUILD_NUMBER} (<${env.BUILD_URL}|Link to build>)"
                            )
                        }                            
                    }           
                }
            }      
        }
    }
}

Mattermost envoie des messages à Jenkins (et reçoit des réponses)

Pour envoyer et recevoir des messages entre Mattermost et Jenkins , il vous suffit d'installer un plugin sur votre serveur Mattermost et de configurer un jeton API sur votre serveur Jenkins.

Les instructions d'installation du plugin Mattermost Jenkins sont assez simples. Mais le processus est devenu encore plus simple avec la nouvelle fonctionnalité de plugins (actuellement en version bêta). Accédez simplement à System Console > Plugins > Jenkins plugin , activez le plugin et entrez votre URL Jenkins.

Vous devez générer une clé API pour permettre à Mattermost de fonctionner avec vos builds Jenkins. Dans Jenkins, cliquez sur votre nom d'utilisateur dans le coin supérieur droit de l'écran, puis accédez à Configurer > API Token et ajoutez un nouveau jeton. Conservez une copie du jeton à utiliser dans Mattermost.

Maintenant, connectez-vous à Jenkins depuis une chaîne Mattermost. Le plugin Jenkins fournit une gamme de commandes slash qui permettent d'interagir avec les tâches Jenkins. 

Une liste complète des commandes peut être consultée ici ou en tapant la commande slash :

/jenkins help

La première commande que vous devez exécuter pour vous connecter est :

/jenkins connect username API_Token

Ensuite, vous pouvez démarrer une build avec :

/jenkins build mattermost-demo

Emballer

Les intégrations permettent à Mattermost d'être bien plus qu'une plateforme de chat. Voici quelques exemples de ce que vous pouvez automatiser :

  • Alertes envoyées lorsque les services sont en panne sur AWS
  • Surveillance d'un cluster Kubernetes
  • Notifications sur les demandes d'extraction de Bitbucket
  • Notifications en temps réel des modifications de configuration Cisco 
  • Rappels de réunion de Google Agenda
  • Notifications Jira et gestion des tickets Jira depuis un canal
  • Transférer les e-mails Microsoft Outlook vers un canal

Comme vous l'avez vu, les intégrations sont un jeu d'enfant à mettre en place !

Ne vous inquiétez pas si un plugin que vous souhaitez n'est pas disponible dans le répertoire d'intégration . La plupart du temps, ce que vous souhaitez peut être réalisé via un simple webhook ou en ajoutant des commandes slash personnalisées. Si vous avez besoin de créer quelque chose de plus puissant, comme un bot, envisagez d'écrire votre propre code qui appelle l'API REST Mattermost. 

Le guide de démarrage mentionné précédemment est très utile. Si vous avez besoin de plus d'aide, il existe également une démo de plugin sur GitHub , qui constitue un excellent moyen d'en apprendre davantage sur la création d'intégrations plus avancées.

Amusez-vous!
